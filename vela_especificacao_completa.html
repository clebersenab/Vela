Vela — Onboarding
do Desenvolvedor

Este documento apresenta o produto, o problema real que ele resolve, a arquitetura definida e o ponto de partida para o desenvolvimento. Leia na ordem — cada seção constrói sobre a anterior.

01

O Problema

Donos de pequenas empresas no Brasil têm contador externo que entrega folha de pagamento e impostos. O que eles não têm é clareza financeira: DRE estruturada, fluxo de caixa real, margens, indicadores. Isso é trabalho de controller — que pequenas empresas não podem pagar.

O resultado é que a gestão financeira fica na planilha. Funciona por um tempo, depois vira um problema que o dono empurra todo mês. Quando o caixa aperta, o problema já começou há três meses.

Por que não usar ERP?

ERP foi feito para equipes maiores operarem. Complexo demais para o dono operar sozinho, caro demais para o ticket do segmento e responde as perguntas erradas. O Granatum é o mais próximo — mas não tem DRE estruturada, nem FCO/FCI/FCF, nem validação fiscal automática.

02

A Solução — Vela

Sistema de controladoria automatizada para pequenas empresas. O usuário lança — o Vela organiza, calcula, confere e apresenta. A IA lê a nota fiscal e preenche os campos automaticamente. A categoria define sozinha se é OPEX ou CAPEX, FCO, FCI ou FCF.

Proposta de valor em uma frase

Contador interno + controller custam a partir de R$15.000/mês e entregam DRE, fluxo de caixa e indicadores. O Vela faz o trabalho dos dois por R$49,90/mês — sem equipe, sem treinamento.

Público: empresas com 1–10 pessoas, faturamento R$20k–R$500k/mês, Simples Nacional ou Lucro Presumido, com contador externo e sem controladoria interna. Usuário típico: o próprio dono ou assistente administrativo.

05

Lógica Fiscal Automática

O sistema gera os lançamentos de imposto automaticamente ao registrar receita. A lógica depende do regime tributário configurado na empresa.

# Lucro Presumido — ao lançar receita de SERVIÇOS: if regime == "LUCRO_PRESUMIDO": criar_pendente("ISS", aliquota=produto.aliquota_iss, vence="dia 10") if produto.pis_cofins == "SIM": criar_pendente("PIS", aliquota=0.0065, vence="dia 25") criar_pendente("COFINS", aliquota=0.03, vence="dia 25") elif produto.pis_cofins == "RETIDO": # IA lê NF e extrai o valor retido criar_pendente("PIS_COFINS_RETIDO", valor=nf_extracted.pis_cofins) if tem_retencao_ir: # ex: Mateus criar_pendente("DARF_IR_RETIDO", valor=retencao_ir, vence="dia 20") if periodicidade_irpj == "TRIMESTRAL": # Gera em julho para o trimestre abr-jun atualizar_provisao_mensal(irpj_estimado / 3) # Simples Nacional: elif regime == "SIMPLES_NACIONAL": criar_pendente("DAS", aliquota=faixa_anexo(receita_acumulada)) # PIS, COFINS, IRPJ, CSLL NÃO gerados separadamente
Regra importante — PIS/Cofins por produto

A isenção de PIS/Cofins é configurada por produto, não pela empresa. O campo products.pis_cofins define se incide (SIM), é isento (NÃO) ou é retido na fonte pelo tomador (RETIDO). O padrão para Lucro Presumido é SIM.

06

Cartão de Crédito — Fluxo Completo

O cartão tem um tratamento especial no sistema. A fatura é importada como PDF, a IA classifica cada linha, e o pagamento é confirmado em lote no vencimento.

# 1. Cadastro do cartão (feito uma vez) cards: { nome: "Cartão XP", dia_vencimento: 20, dia_fechamento: 10 } # 2. Importação da fatura (todo mês) usuario → sobe PDF da fatura IA → lê linha por linha → para cada item: - reconhece → sugere categoria → usuario confirma - não reconhece → usuario classifica - parcelado → lança só parcela atual (futuras entram nas faturas seguintes) → cada item = entry com status PENDENTE + card_id + forma_pagamento: CARTAO # 3. Agrupamento no Contas a Pagar CP exibe: "Fatura Cartão XP — R$ 3.601,93 — vence 20/07" # 4. Pagamento em lote (dia 20) usuario → "Confirmar pagamento da fatura" → todos os entries da fatura: status PENDENTE → PAGO → entram na DRE e DFC automaticamente

M3

Produtos e Serviços

Cadastro de produtos e serviços da empresa. Define tipo, alíquotas e — criticamente — a incidência de PIS/Cofins, que varia por produto.

MVP
UC05 Cadastrar Produto ou Serviço MVP
Ator
Usuário autenticado
Descrição
Cadastro com definição de tipo, alíquotas fiscais e incidência de PIS/Cofins. A incidência de PIS/Cofins é configurada por produto — não pela empresa. O padrão para LP é SIM, mas produtos específicos podem ser isentos ou com retenção na fonte.
Fluxo
1. Produtos → Novo → 2. Preenche: Nome · Tipo (PRODUTO | SERVIÇO) · Alíquota ISS (%) · PIS/Cofins: SIM (0,65%+3%, padrão LP) | NÃO (isento) | RETIDO (IA extrai da NF) · Custo Direto (%) → 3. Salva.
UC06 Listar e Editar Produtos MVP
Ator
Usuário autenticado
Descrição
Lista todos os produtos/serviços com filtro por tipo. Permite editar alíquotas e incidência fiscal.
Fluxo
Produtos → Lista com tipo, alíquotas e status. Clica em editar → altera campos → salva. A mudança de alíquota vale para lançamentos futuros, não retroativos.
Regra de PIS/Cofins por produto

A isenção de PIS/Cofins não é da empresa — é do produto. A CBS Participações é isenta nos produtos digitais (e-books, cursos) mas nos serviços o PIS/Cofins é retido na fonte pelo tomador. Configure campo a campo no cadastro.

M4

Plano de Contas

54 categorias pré-configuradas amarradas aos níveis da DRE e ao fluxo de caixa. O usuário nunca escolhe categoria — digita a descrição e a IA categoriza.

MVP

A categoria define automaticamente onde o lançamento aparece na DRE e a qual seção do DFC pertence (FCO, FCI ou FCF). O campo natureza_fluxo na tabela entries é definido automaticamente pela categoria no momento do lançamento.

Regra de ouro

O usuário nunca vê códigos de categoria. Digita a descrição → IA sugere → confirma com um clique. O sistema aprende o padrão — da segunda vez em diante já categoriza sem perguntar.
